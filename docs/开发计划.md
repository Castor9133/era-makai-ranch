## 1. 丰富「拜访」玩法（访问线）

### 1.1 新增：拜访-轻量 H 互动（不进入完整调教）
- **定位**：与“摸头/接吻/吸欧派/灵力注入/投药”同层，短平快、带少量收益/风险。
- **候选玩法**（建议先做前 3 个）：
  - **揉胸/吸乳**（非授乳）：少量快感 + 轻微好感/屈辱分岔（按陷落/性格）。
  - **[18] 亲热一下**（手口足、结尾可选射精）：流程末由玩家选择**口内射精**或**忍住不射**；不射走精力/好感/尊严等轻结算，射精走咽精与更强数值；需恋慕或隶属且奴隶**事先穿好上下内衣**（街市购买成人内衣后在更衣装备），强调羞耻感。
  - **强制拥抱/贴身嗅闻**：偏精神与标记（见“标签线”）。
  - **拜访-情趣内衣门槛**：不再用一次性润滑/媚药小道具；以「已装备上下内衣」作为侍奉类短互动的前置，与购买—更衣—羞耻叙事一致。

### 1.2 访问线结构优化（便于扩展）
- **把每个拜访玩法抽成独立 @ 子程序**：例如 `@VISIT_ORAL_TEASE`、`@VISIT_HANDJOB`。
- **统一时间限制**：沿用现有 `CFLAG:时刻限制／…` 命名。
- **统一“执行后收尾”**：`SOURCE_CHECK`、必要时 `COM890`、`PRINTW` 与 `SELECTCOM` 复位，避免分支漏掉。

### 1.3 访问-UI 体验
- **在菜单上显示冷却/条件原因**（灰字）：缺精力/灵力/道具/处女结界/当日已做等。
- **把“失败原因”文本规范化**：保证同一原因在不同玩法里提示一致。

### 1.4 验收点
- 能连续执行多个拜访玩法，不出现 `SELECTCOM` 残留、射精/体位标志混乱、或无法返回菜单的问题。
- 口上至少：泛用口上对新增拜访分支有一句非兜底台词；否则列欠番清单。

## 2. 丰富「标签」的作用（状态/标记线）

### 2.1 先定“标签”范围与用途（建议两类并行）
- **行为标签**：描述近期行为/偏好（如：爱口/厌肛/渴求注入/怕洗澡）。
- **关系标签**：描述对 PC 的位置（如：访客亲昵、把你当主人、把你当工具）。

### 2.2 标签如何影响系统（建议按“轻→重”迭代）
- **轻影响（第一版就做）**
  - 影响拜访菜单可选项（解锁/加权）。
  - 影响口上分支（在 `KOJO_PRINT("访问", …)` 或调教命令口上里读标签）。
  - 影响 `SOURCE_UP` 的小修正（例如屈辱/爱情/支配微调）。
- **重影响（第二版）**
  - 影响战败捕获后“处理方式”选项（见第 4 部分）。
  - 影响洗澡事件（羞耻/抵抗/顺从）。

### 2.3 数据落点（建议）
- **短期标签**：用 `TFLAG`（回合/当天清理）。
- **中期标签**：用 `CFLAG`（持久，带衰减计时：剩余日数）。
- **长期标签**：用 `MARK`/`ABL` 或专用 `CFLAG:标签_…`（慎用，避免膨胀）。

### 2.4 验收点
- 标签的新增/清理路径清晰：何时设置、何时衰减、何时清零。
- UI 或提示能让玩家知道“为什么这个选项变灰/变亮”。

## 3. 丰富「洗澡」相关玩法（夜间/日程线）

### 3.1 玩法方向（建议先做事件化）
- **洗澡事件池**：按状态触发不同事件（污渍、精液、药效、怀孕/产褥、口枷等）。
- **洗澡交互**：
  - 帮洗（温柔） vs 强制冲洗（羞辱）
  - 洗前检查（标签/道具/怀孕状态提示）
  - 洗后奖励/惩罚（体力、心情、清洁度、屈辱）

### 3.2 与系统联动
- **清洁/污渍系统**：洗澡可清掉特定 `TFLAG`/污渍标志；某些污渍清理需要道具。
- **标签联动**：怕洗澡/喜欢被看/喜欢被摸等，影响台词与数值。

### 3.3 验收点
- 洗澡流程不破坏夜间回合结算；不与妊娠日结、绝顶育成日结冲突。

## 4. 丰富「奴隶战败捕获」玩法（战斗→处置线）

### 4.1 捕获后的“处置”分层
- **立即处置**（战后当场）：束缚、验身、标记、简单治疗/止血、封口。
- **押送/登记**：入栏、编号、初始标签写入（例如“仇恨/恐惧/抵抗”）。
- **首夜事件**：根据标签与性格触发不同的“破防/试探/反抗”剧情入口。

### 4.2 与现有系统对接（建议先做钩子）
- 找到战败捕获的主入口（战斗结算/生成奴隶处）并加 `TRYCALL` 钩子：
  - `CAPTURE_HOOK_POST_CREATE`（已生成角色卡后）
  - `CAPTURE_HOOK_POST_ASSIGN`（分配栏位/状态后）
- 钩子里只做：写标签/写初值/安排后续事件，不要塞长剧情，避免难维护。

### 4.3 验收点
- 捕获后不会出现“状态不完整导致口上/调教报错”。
- 新增处置选项与标签能在后续拜访/洗澡里体现。
