- **权力差**：感恩、救助、同住缩小心理距离；**自卑**与「值不值得被接纳」推动献身冲动。
- **称谓**：由疏远称呼向直呼名过渡可作为**段落台阶**。
- **催化剂**：仪式感（生日等）、饮酒、脆弱时刻；男视角可带**生活细节观察**。
- **身体描写粒度**：从衣料、呼吸、距离感入手，适合 **拜访短时轻 H**。

## 口上与玩法扩展同步规则

本文档约定：**凡新增或改动与调教相关的道具、指令（COM）、体位、`TRAINNAME`、射精／绝顶分支、地文入口等，须同步检查并更新所有相关口上**；实现不了或拿捏不准时，**明确交回给人写**。

### 一、为什么要同步

口上依赖 **`KOJO_PRINT` 的第一、第二参数**（如 `"调教命令"` + `TRAINNAME:SELECTCOM`）和 **`SELECTCASE ARGS:1`** 里的**指令名字符串**。
系统侧若新增指令名、改名、或增加 `SOURCE:`／`TEQUIP:` 分支，而口上仍只有 **`CASEELSE`** 兜底，会出现：

- 只有通用一句，**性格差消失**；或
- 某些分支**永远不触发**（拼写与 `Train.csv` / `TRAINNAME` 不一致）。

因此：**玩法与口上视为同一次交付**，除非刻意声明「本版不写口上」。

### 二、系统侧改动时，口上要查哪些地方

按改动类型勾选（不必每次全做，但**相关项必须想到**）。

| 改动类型 | 建议检索 / 更新位置 |
|----------|------------------------|
| **新调教指令或改名** | `CSV/Train.csv`、`TRAINNAME:SELECTCOM`；所有 **`ERB/○口上/汎用口上/*_調教コマンド.ERB`** 里 `SELECTCASE ARGS:1` / `GROUPMATCH`；**单文件口上**（如 `懦弱口上1.ERB`）内 `@*_TRAIN`；**`○動作確認用口上_調教コマンド.ERB`**（若有新 CASE 示范）。 |
| **新道具 COM200+ 等** | 地文 `メッセージ_COM*.ERB`；口上同上 + **`ARGS:2`「执行／移除」** 是否需新分支。 |
| **新体位 / 插入位** | `Train.csv` 名称与口上 `CASE` 列表一致；`SOURCE_UP` / 射精若新 `SOURCE:`，检查 **`メッセージ_射精.ERB`** 与各口上 **`CASE "射精"`**。 |
| **新射精方式 / 新 `SOURCE:` 旗标** | `○射精処理.ERB`、`メッセージ_射精.ERB`；各泛用口上 **`CASE "射精"`** 内 `IF SOURCE:…`。 |
| **新绝顶 / 新 `NOWEX`** | 口上 **`CASE "绝顶"`**；触手侧另看 **`メッセージ_触手射精.ERB`** 是否与 `KOJO_PRINT` 联动（当前触手射精口上常注释，若开启要一起补）。 |
| **新日程 / 事件 / 战斗 / 访问** | `KOJO_PRINT` 第一参数；各 **`CASE "日程"` / `"事件"` / `"战斗"` / `"访问"`** 等。 |
| **专用口上** | `ERB/○口上/[オリジナル]…`、`[era魔界牧場]…` 下对应 **`EX_KOJO_*`**；规则与泛用相同。 |

**检索技巧**：在 `ERB/○口上/` 下对**新增指令的日文/中文名**、`SELECTCOM` 号段、或 `CALL KOJO_PRINT` 新字符串做全文搜索，避免漏文件。

### 三、交付清单（给改代码的人 / AI）

完成「新玩法」相关合并前，尽量满足：

1. [ ] **Train / COM / 地文** 已与现有工程命名一致，可实际跑通。
2. [ ] **所有泛用口上**：至少在新指令上有一条**非兜底**台词，或**有意识扩展 `CASEELSE`** 的叙述（避免全角色同一句话）。
3. [ ] **单文件范本口上**（当前为 **`懦弱口上1.ERB`**）：`@*_TRAIN` 内对新指令有 `CASE` 或更新兜底说明。
4. [ ] **动作确认用口上**（若项目依赖它做回归）：能覆盖新指令则更好。
5. [ ] 若本轮**不写口上**：在 PR 说明或本目录 `docs` 里记一条 **「口上欠番：待补」**，并列出**指令名列表**，方便你或后续作者补。

**汎用調教分册（`*_調教コマンド.ERB`）与 `CSV/Train.csv` 显式 CASE 对照表**：见 [`docs/汎用調教分册_Train欠番表.md`](./汎用調教分册_Train欠番表.md)，可用根目录命令 `python tools/diff_train_kojo_generic.py` 重新生成。

### 四、何时把口上交回「人」写

出现以下情况时，**不要在口上里硬编敷衍长文**；应在本文件或 PR 中列出 **「须补口上清单」**（指令名 + 建议性格口吻），交给你写：

- 需要**强角色性格 / 梗 / 剧情连贯**的长段落，上下文不足容易 OOC；
- 涉及**版权角色专用口上**、或你**个人口味固定**的台词；
- 新增分支与现有 **`＃陷落状态…＃` / 刻印 / 素质** 组合过多，AI 无法保证**全组合不重复、不踩雷**；
- 时间不够：宁可 **只改系统 + 地文 + `CASEELSE` 一行占位**，把清单交你补全。

你在清单里可写：「指令 X → 泛用全部 `CASE` + 懦弱单文件 + 彩华专用」等优先级。

### 五、与 Cursor / Agent 的协作话术（可复制）

*维护：魔改仓库；新增泛用口上文件时，把第二节表格里的路径补全即可。*

## 口上创作 DEMO 与性格扩展清单

本文档供你**自写多套泛用口上**时使用：前半是**工程内接法**与**最小可跑 DEMO**；中段是**与常见二次元游戏相比容易缺的 archetype**；末段是**推荐 8～10 个新「性格」词条**（写入 `Str.csv` 3000 段 + 随机性格池 + 可选口上套装名）。

**与系统同步的硬性约定**（新道具 / 指令 / 体位等改代码时口上必须跟）：见**上文「口上与玩法扩展同步规则」**。AI 或协作者若写不好口上，应列出欠番清单交你补写。

### 一、本工程里「性格」和「口上」的关系（必读）

| 概念 | 存哪 | 作用 |
|------|------|------|
| **性格** | `CSTR:性格`（文字，如「强势」） | 随机素質、称呼等**数值/逻辑**会参考；**不自动绑定**口上。 |
| **使用口上** | `CSTR:使用口上`（如「強気1」） | `KOJO_PRINT` → `TRYCALLFORM KOJO_GENERAL_INDEX_%CSTR:使用口上%`。 |
| **口上显示** | `CFLAG:口上显示`、`FLAG:口上显示` | 关则整局不显示。 |

**泛用口上**要同时做三件事：

1. 新建 `ERB/○口上/汎用口上/〇〇口上1.ERB`（或你起的套装名），内含 **`@KOJO_GENERAL_INDEX_口上識別名(ARGS:0,ARGS:1,ARGS:2,ARGS:3)`**。
   - 函数名**必须**与 `CSTR:使用口上` 完全一致（例：`KOJO_GENERAL_INDEX_懦弱1`）。
2. 在 **`CSV/Str.csv` 的 6100～** 增加一行：**编号,口上識別名**（与上面函数后缀一致）。
3. 在游戏内「口上设置」里能枚举到该函数（`ENUMFUNCWITH "KOJO_GENERAL_INDEX_"`）。

台词用 **`CALL MESSAGE_BOX,@"……"`** 放入候选箱，由 **`MESSAGE_PRINT`** 随机抽一条显示。
换行用 **`\n`**（见 `○口上COMMON.ERB` 注释）。

**汉化向约定**：玩家在游戏里看到的口上正文，建议**统一写简体中文**；`CASE "……"` 里的指令名、`%MASTER_CALL()%` 等占位符仍须与工程内日文标识一致，勿改。

### 二、`KOJO_PRINT` 常见第一参数 `ARGS:0`（写作目录）

下列来自仓库内 `CALL KOJO_PRINT(...)` 的整理，**新口上至少建议先覆盖加粗**的高频项；其余可 `CASEELSE` 留空或接共通默认。

| ARGS:0 | 说明（第二、第三参数常见形态） |
|--------|----------------------------------|
| **调教命令** | `ARGS:1` = 指令名（`TRAINNAME:SELECTCOM`）、`ARGS:2` = `"执行"` / `"移除"` 等 |
| **射精** | 常与 `SOURCE:膣内射精经验` 等配合在口内分支写 |
| **绝顶** | 高潮时 |
| **日程** | 如 `"调教","开始调教"` / `"调教结束"` |
| **事件** | 如领取奴隶、母乳体质化等 |
| **战斗** | 攻击、被击中、战斗开始等 |
| **访问** | 入口、改称呼、改发型等 |
| **吸欧派** | 开始 / 结束 |
| **例文** | 口上设置界面展示用，**建议必写**一句自我介绍 |

更多分支可在现有 **`汎用口上/*口上1.ERB`** 里搜 `SELECTCASE ARGS:0` / `GROUPMATCH(ARGS:1` 照抄结构。

### 二点五、单一文件整套范本（推荐直接复制改）

仓库内已提供 **完整一套「懦弱」泛用口上**，全部写在一个 `.ERB` 里（含 `TRAIN`），作为生产其他性格的母版：

- **`ERB/○口上/汎用口上/懦弱口上1.ERB`**
- **`Str.csv`** 已登记 **`6110,懦弱1`**

游戏内把奴隶 **`CSTR:使用口上`** 设为 **`懦弱1`** 并打开口上显示即可测试。复制后全局替换 **`懦弱1` / `懦弱口上1` / `KOJO_GENERAL_INDEX_懦弱1` / `KOJO_GENERAL_懦弱1_TRAIN`** 等标识即可套壳写新性格。

### 三、最小可跑 DEMO（复制后改名字）

**文件**：`ERB/○口上/汎用口上/DEMO性格口上1.ERB`（文件名随意，但**索引名**要与 `Str.csv` 一致）

```eramaker
; ======================================
;口上識別名：DEMO性格1
;口上作者：（你的名字）
;许可证：CC BY-NC（与现有泛用口上统一即可，或自填）
; ======================================

@KOJO_GENERAL_INDEX_DEMO性格1(ARGS:0,ARGS:1,ARGS:2,ARGS:3)

SELECTCASE ARGS:0
CASE "例文"
	CALL MESSAGE_BOX,@"「这里是【DEMO性格1】口上样例。\n陷落前后、命令与绝顶会按性格分叉。」"

CASE "调教命令"
	; 最简：不区分指令，先写通用品
	IF ARGS:2 == "执行"
		CALL MESSAGE_BOX,@"「……（执行 %ARGS:1% 时的台词）」"
	ELSEIF ARGS:2 == "移除"
		CALL MESSAGE_BOX,@"「……（移除装备时的台词）」"
	ENDIF

CASE "绝顶"
	CALL MESSAGE_BOX,@"「……要去了……」"

CASE "射精"
	IF SOURCE:膣内射精经验
		CALL MESSAGE_BOX,@"「……里面……」"
	ELSE
		CALL MESSAGE_BOX,@"「……射在外面……」"
	ENDIF

CASEELSE
	; 未实现的分支留空即可
ENDSELECT
RETURN
```

**`Str.csv`（6100 段附近）追加一行**（编号不与现有重复即可）：

```csv
6110,DEMO性格1
```

游戏内把该奴隶 **`CSTR:使用口上`** 设为 **`DEMO性格1`**，打开口上显示，执行任意带 `KOJO_PRINT("调教命令",...)` 的指令即可测。

**口枷 / 深喉等**：多数泛用口上在 `TFLAG:Ｍ装备` 时 `CALL KOJO_GENERAL_MOUTH_SHACKLE_1` 直接 `RETURN`；你可在 DEMO 后期再接同一套路，避免开局工作量爆炸。

### 四、和「一般手游 / 拔作」比，现有泛用口上**容易缺**的维度

当前仓库泛用套装大致覆盖：**强气、元气、品、姐系、认真、堅物、无口、幼め非敬语、男孩子气、慇懃**。
相对**常见卡池**仍容易缺（你举的例子落在这些里）：

| 缺口类型 | 玩家直觉标签 | 备注 |
|----------|--------------|------|
| **低反抗 / 高顺从** | 懦弱、胆小、受け身 | 与「弱势」不同：弱势可偏傲；**懦弱**更偏「不敢拒绝、道歉多」。 |
| **和风母性 / 外柔内刚** | 大和抚子 | 与「温厚」「気品」有重叠，缺的是**敬语 + 献身 + 羞耻的拧巴**。 |
| **认知差** | 天然呆、小妹妹 | 「元気っ子」偏吵；**天然**偏慢半拍、自爆；**妹系**偏依赖称呼与撒娇语法。 |
| **腹黑 / 算计** | 毒舌笑面、小恶魔 | 「傲慢」不够「**笑着把你卖了**」。 |
| **丧 / 虚无** | 阴角、厌世 | 与无口不同：无口可寡言；**丧**是**否定句、自虐、兴趣稀薄**。 |
| **秩序恶 / 乐子人** | 混沌、挑事 | 强气偏对抗；**乐子人**偏**围观自己受害**或**挑衅主人**。 |
| **成熟色气** | 妖艳、痴女系姐姐 | 「姉御」偏强；缺**主动勾引、慢节奏荤话**。 |
| **常识人吐槽** | 正常人被迫害 | 适合喜剧调剂；与「生真面目」不同：**吐槽役 + 被迫参与**。 |

写口上时可自检：**陷落前 / 屈服 / 恋慕 / 隶属 / 崩坏** 是否各有一套语气；**处女・恋慕・害羞** 等素质是否做 `GROUPMATCH` 微分。
