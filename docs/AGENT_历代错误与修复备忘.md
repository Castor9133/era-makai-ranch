# era-makai-ranch：历代错误与修复备忘（供后续 Agent 学习）

本文档汇总 **本仓库在 Cursor 协作中已出现并已修复（或已约定规避）** 的典型问题，便于后续 Agent 快速对齐工程习惯、避免重复踩坑。  
**维护约定**：每次由 Agent 或人工修复一类新错误后，在文末「修订记录」追加一行日期与摘要，并在上表或分节中补一条。

---

## 一、速查表（症状 → 方向）

| 症状 / Emuera 报错 | 常见根因 | 优先检查 |
|--------------------|----------|----------|
| `配列変数"XXX"の第1引数(N)は配列の範囲外` | `#DIM` 长度与 `FOR …,0,6` 或 `:5` 等下标不一致 | 声明处 `.ERH`、循环与所有 `XXX:数字` 用法 |
| 未定义变量 / 颜色不生效 | **中文标识** 与 `FLAG.csv` / `Str.csv` 等 **一字不一致**（「表示」vs「显示」） | `CSV/` 与 `FLAG:` 引用全文检索 |
| `ENDIF` 不匹配、解析失败 | 口上或长 `IF` 链漏写 `ENDIF`、或 `IF NOWEX:5` 等与引擎分支混用 | 从报错行向上配对 `IF`/`SELECTCASE` |
| 妊娠补丁不加载 / 宏无效 | `#DIM` 写在 **非 `.ERH` 的 ERB 文件首**（Emuera 规则） | 全局量集中到 `MAKAI_PREGNANCY_GLOBALS.ERH` 等 |
| 子数组访问越界或逻辑错 | 二维数组仍用 `魔界_子日:LOCAL` 而 `LOCAL` 在嵌套里被改写 | 改为 `:(LOCAL:0)` 等 **显式槽位** |
| `関数"@COMxxx"が見つかりません` | **`Train.csv` 已登记 xxx**，但 `○TRAIN/COMMAND` 无 `@COMxxx`**（常有 `_MESSAGE` / `EQUIP` 却无本体） | 全文检索 `@COM` + 编号；见下文 **§6 Train 与 COMMAND 突合** |
| 缺 **A爱抚**／**臀部素股** 导致手感与射精分类乱跳 | 新 COM 未对齐既有 **玩弄阴唇**、**素股** 的数值与 `Ｐ使用`／`○射精処理` 约定 | 见下文 **§8**；新增后做 Train↔COMMAND↔射精链路与足交/口内优先级自检 |

## 二、分条说明（含路径与修复要点）

### 1. `EXPERIENCE_FIX` / `SENSITIVE_FIX` / `MARK_FIX` 数组越界

- **现象**：调教中（如揉胸）触发 `SOURCE_CHECK` → `SENSITIVE_FIX_CALC` 时崩溃：  
  `○調教パラメータ計算\○快楽倍率処理.ERB` 第 92 行附近，`EXPERIENCE_FIX:LOCAL` 报错 **第 1 引数 (5) 超出范围**。
- **根因**：`@SENSITIVE_FIX_CALC_EXPERIENCE` 使用 `FOR LOCAL,0,6`，即 `LOCAL ∈ {0,1,2,3,4,5}`（共 6 段，对应部位感覚等）。  
  另：`○快楽倍率処理_刻印等.ERB` 等处使用 `SENSITIVE_FIX:5`、`MARK_FIX:5`。  
  若 `#DIM EXPERIENCE_FIX,5`（Emuera 常见语义为 **下标 0～4 共 5 元素**），则 `LOCAL=5` 非法。
- **修复**：在 `ERB/●ERH/非保存広域変数.ERH` 将下列三项由 `,5` 改为 `,6`，并与注释中「0～5：Ａ/Ｂ/Ｃ/Ｖ/Ｎ/Ｍ」一致：  
  - `#DIM EXPERIENCE_FIX,6`  
  - `#DIM SENSITIVE_FIX,6`  
  - `#DIM MARK_FIX,6`
- **给 Agent 的规则**：凡 `FOR LOCAL,0,N` 写入 `ARR:LOCAL`，`#DIM ARR` 的第二参数须 **≥ N**（按所用 Emuera 版本手册确认是「元素个数」还是「最大下标」）；同时全局搜 `ARR:` 避免漏改。

---

### 2. 拟音颜色：`SETCOLOR` 与 `FLAG.csv` 不同名

- **现象**：拟音相关颜色不生效或报未定义（取决于引擎与检查方式）。
- **根因**：`表示関数\○擬音表示.ERB` 曾使用 `FLAG:拟音表示颜色编码`，而 `CSV/FLAG.csv` 登记名为 **`拟音显示颜色编码`**（「表示」与「显示」混用）。
- **修复**：`○擬音表示.ERB` 第 6 行等处改为 `SETCOLOR FLAG:拟音显示颜色编码`；配置界面 `●CONFIG\表示設定\擬音表示設定.ERB`、`●CONFIG\◯デフォルト\デフォルト設定_表示.ERB` 等与 CSV 保持一致。
- **给 Agent 的规则**：改 `FLAG:` / `TFLAG:` / `STR:` 名时 **必须以对应 CSV 行为准**，全仓库 `grep` 旧名后再提交。

---

### 3. 妊娠 / 托儿所扩展（魔界线）集成类错误

以下来自会话中对 **妊娠补丁落地** 的整理；实现以当前 `ERB/○妊娠/` 与 `PREGNANCY_集成说明.txt` 为准。

| 问题 | 处理方式 |
|------|----------|
| 在 `○PREGNANCY_CORE.ERB` / `○NURSERY_CORE.ERB` **文件开头**写 `#DIM` 等 | Emuera 要求：普通 ERB 顶不宜作全局声明；改为 **`ERB/MAKAI_PREGNANCY_GLOBALS.ERH`** 集中 `#DEFINE` / `#DIM` / `SAVEDATA`，与 ERB 同目录由引擎加载（见集成说明第二节） |
| `@PREGNANCY_DEBUG_*` 等宏缺少 `ARG:0` | 按调用约定补全参数，避免未定义实参 |
| `PREGNANCY_ABORT_MAKAI` 等 `IF` 链缺 `ENDIF` | 配对块结构，跑一遍编译/执行路径 |
| `魔界_子日` / `魔界_子生母` 一维下标在嵌套循环中错位 | 使用 `魔界_子日:(LOCAL:0)` 等形式，**避免与外层 `LOCAL` 混淆** |
| 文档与常量不一致 | 以 `ERB/○妊娠/PREGNANCY_集成说明.txt` 与 `MAKAI_PREGNANCY_GLOBALS.ERH` 为单一事实来源；改日数/槽位须同步托儿所逻辑 |

### 4. 泛用口上与脚本卫生（摘要）

- **ENDIF / 块结构**：长口上 `IF`/`SELECTCASE` 易漏 `ENDIF`，Emuera 报错行号常指向**更深处**，需从报错点向上配对。
- **`NOWEX:5` 等**：与尿道等扩展绝顶分支并存时，注意与 `NOWEX:总绝顶` 等逻辑是否重复或漏判；修改后跑一条相关 COM 验证。
- **字符串与编码**：全角 `％`、不可见字符、错误转义会导致显示异常或解析错误；口上正文换行约定见 `docs/口上写作指南.md`。
- **同步规则**：调教相关新增/改名必查 `docs/口上与玩法扩展同步规则.md`。

---

### 5. 战斗与其它 `TFLAG` / `CFLAG` 命名

- **原则**：ERB 中 `TFLAG:xxx` / `CFLAG:xxx` 必须与 **`CSV/TFLAG.csv`、`CSV/FLAG.csv`** 中登记名 **完全一致**（含简繁、下划线）。
- **历史摘要**：曾出现「魔物攻击」类 ERB 使用与 `TFLAG.csv` 不一致的 `TFLAG:` 名（如口上开关类变量），已按 CSV 对齐。若后续报错「未定义」，先 CSV 再 ERB。

---

### 6. `Train.csv` 与 `@COM` 欠番（例：`@COM59` 足交）

- **现象**：运行时报 **`@COMxxx` が見つかりません**。菜单里的 **`[47]`** 常为显示序号，`SELECTCOM` 可能是 **59**（足交）等，勿混淆。
- **突合**：从 `CSV/Train.csv` 收集指令编号；在 `ERB/○TRAIN/COMMAND/` 全文检索行首 **`@COM` + 编号**。Train 有而 COMMAND 无 → 缺 `@COMxxx`（也可能仅有 `_MESSAGE` / `EQUIP_*` 而无本体）。
- **`Tequip` 与継続**：`○調教パラメータ計算\○装備SOURCE増加\○装備SOURCE増加COMMON.ERB` 中 **`FOR LOCAL,0,300` + `TEQUIP:LOCAL`** 会 **`CALLFORM EQUIP_{LOCAL}_CONTINUE`**，故 **`Tequip.csv` 左列编号**必须存在对应的 **`@EQUIP_n_CONTINUE`**（必要时再加 `_MESSAGE`）。注意：**Train の SELECTCOM と Tequip 左列不一定同名同号**（例：`COM214` 装着尿道塞对应 **`EQUIP_215_CONTINUE`**；**肛门拉珠**虽为 Train **215**，在本工程中 **`Tequip.csv` 使用 `234,肛门拉珠`**，継続为 **`EQUIP_234_CONTINUE`**）。
- **2026-05 批量**：已补齐一度欠番的 **38／48／59／71／72／116／117／210／211／215**（含 `Tequip`、`EQUIP`、地文、素質 **`669` 榨乳嗜好**中对 **117／乳腺侵入触手** 的引用）。日后新增 Train 行务必同步 COMMAND 与 Tequip／EQUIP。

---

### 7. `○PREGNANCY_CORE.ERB` 文件首行 `0;`

- **现象**：解釈できない行／関数定义前有非法语句。
- **原因**：注释写成了 **`0;`**，emuera 会把 **`0`** 当表达式解析。
- **修复**：改为 **`;`** 开头的普通注释行。

---

### 8. 欠规划：`A爱抚`／`臀部素股`（应对齐既有 COM）

- **现象**：无独立 **A爱抚**（肛门／会阴侧**非插入**的爱抚、抚弄）、**臀部素股**（臀缝／臀腿间**非插入**的素股式摩擦）时，若随手新写一套 SOURCE／射精分支，容易与全工程 **玩弄阴唇**、**素股** 的刻度不一致，或复现 **陈旧 `Ｐ使用` 位** 误判（参见修订记录中足交／口内／乳交类修复）。
- **落地原则**：**以现有 COM 为单一事实来源做复刻或等比改写**，勿从零自定一套。
  - **`A爱抚`** → 对照 **玩弄阴唇**（或工程内等价「外阴非插入玩弄」指令）：SOURCE 强弱、屈辱／性趣比例、习熟与经验落点改为 **Ａ向**（Ａ感度、肛门爱抚／肛门经验等以你工程 `exp.csv`/`ability` 为准）；`TEQUIP` 与 **`TEQUIP_ADJUST`/`Ｐ使用`** 与模板 COM 一致——若模板不占阴茎插入位，A爱抚也应避免**误挂**插入类位导致走**宫内／口内**射精。
  - **`臀部素股`** → 对照 **素股**：**非插入**摩擦；`○射精処理` 里与素股 **同档优先级**（必要时与足交修复同理：**专用 `TEQUIP` 或体位分支先于** 泛用 `Ｍ插入`／**乳交** 判定）；若有 **公开羞耻** 类 `TFLAG`，素股怎么写，臀部素股就怎么对齐或并列扩展。
- **验收**：`Train.csv` ↔ `@COMxxx` ↔ `MESSAGE`／`EQUIP_*` ↔ 射精选项与地文类别一致；口上按 `docs/口上与玩法扩展同步规则.md` 增补 `KOJO_PRINT` 分支 key。

## 三、推荐工作流（Agent）

1. 用户粘贴 **完整 Emuera 报错**（含文件路径、行号、栈）时，先打开该行及 **声明文件（多为 `●ERH/*.ERH`）**。  
2. 任何 `FLAG:` / `TFLAG:` 报错：在 `CSV/` 中 **精确匹配** 字符串。  
3. 数组越界：对照 **`#DIM` 第二参数语义** + 所有 `FOR` 上界 + 手写下标 `:N`。  
4. **`@COM` 找不到**：做一次 **Train.csv ↔ COMMAND** 编号突合（见上节 §6）。  
5. 改口上或地文：遵守工作区规则——**玩家可见口上简体中文**；引擎标识符与 CSV 一致。  
6. 大功能补丁：在 `docs/` 增加或更新 **集成说明**（本文件不替代专项说明）。
